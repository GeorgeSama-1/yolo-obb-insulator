# Stage1 YOLO-CLS Patch Pipeline Design

**Date:** 2026-04-17

## Goal

Build a practical Stage 1 two-stage data-preparation and training path that:

- crops patch images directly from the strongest Stage 2 dataset variant
- keeps all `normal` patches
- oversamples only `abnormal` patches until the train split is close to `2:1`
- trains a `YOLO-cls` patch classifier at `384`

In the same round, add a convenience script for `stage2_abn_boost_x6`.

## Decisions

### Patch source

Use:

- `data/processed/stage2_defect_obb_abn_boost`

as the default Stage 1 patch source.

### Patch labels

Each Stage 2 OBB annotation becomes one patch:

- `normal`
- `abnormal`

### Balancing strategy

Train split only:

- keep all `normal`
- oversample only `abnormal`
- target approximately `normal:abnormal = 2:1`

Validation split:

- keep original distribution

### Classifier

Use:

- `yolo11m-cls.pt`
- `imgsz=384`

## Outputs

### Stage 1 patch dataset

- `data/processed/stage1_patch_classifier/`

with layout:

- `train/normal`
- `train/abnormal`
- `val/normal`
- `val/abnormal`

### Stage 2 convenience dataset boost

- `data/processed/stage2_defect_obb_abn_boost_x6/`

generated by a dedicated helper script.

## Implementation units

- `src/stage1_two_stage/patch_dataset.py`
  - crop records from Stage 2 YOLO OBB data
  - convert OBB polygons to axis-aligned patch bounds
  - write patch classifier dataset
  - oversample `abnormal` train patches only

- `scripts/data/prepare_stage1_patch_classifier.py`
  - CLI for Stage 1 patch dataset generation

- `src/stage1_two_stage/classifier_train.py`
  - add Stage 1 train args builder for `YOLO-cls`

- `scripts/train/train_stage1_classifier.py`
  - real Stage 1 training entrypoint

- `scripts/data/prepare_stage2_abn_boost_x6.py`
  - convenience wrapper for `x6`

## Validation

- tests for patch dataset generation
- tests for oversampling behavior
- tests for Stage 1 train arg construction
- docs update for Stage 1 workflow and x6 helper usage
